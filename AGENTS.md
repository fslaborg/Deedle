# Deedle – Agent Guide

## Repository Overview

Deedle is an F# library for data manipulation providing efficient frame and series data structures (similar to Python's pandas). It supports missing values, aggregations, grouping, joining, statistics, and time-series alignment.

Key sub-projects:

| Project | Path |
|---|---|
| Core library | `src/Deedle/` |
| .NET Interactive formatting | `src/Deedle.Interactive/` |
| MathNet.Numerics integration | `src/Deedle.Math/` |
| Excel integration | `src/Deedle.Excel/` |
| R type-provider plugin | `src/Deedle.RProvider.Plugin/` |

Tests live under `tests/`.

---

## Prerequisites

### .NET SDK
The pinned version is **5.0.103** (`global.json`). The `rollForward` policy is `major`, so a newer SDK will be accepted, but 5.x is what CI uses. Install from <https://dotnet.microsoft.com/download>.

### R (required for the `Deedle.RProvider.Plugin` project and the default `All` build target)
- R 4.0.2 or compatible installation.
- The `zoo` CRAN package must be installed:
  ```r
  install.packages('zoo')
  ```
- The following environment variables must be set before running the build:
  ```bash
  export R_HOME=$(R RHOME)
  export R_LIBS_USER=~/.local/lib/R   # Linux only
  ```
  On Windows, `R_HOME` is set analogously; `R_LIBS_USER` is not required.

If R is not available, use the `AllNoR` target (see below) to skip the R plugin.

## Setup (one-time after cloning)

```bash
# 1. Restore .NET local tools (includes FAKE and Paket)
dotnet tool restore

# 2. Restore NuGet packages via Paket
dotnet paket restore
```

Both steps are required before any build or test command.

## Building and Testing

The project uses [FAKE](https://fake.build/) as its build system, driven by `build.fsx`.

### Full build + tests (what CI runs)

```bash
dotnet fake build
```

This runs the default `All` target, which chains:

```
Clean → AssemblyInfo → Build → BuildTests → RunTests → All
```

- **Build**: compiles all five source projects in `Release` configuration with SourceLink enabled.
- **BuildTests / RunTests**: builds and runs the NUnit test projects:
  - `tests/Deedle.Tests/`
  - `tests/Deedle.Math.Tests/`
  - `tests/Deedle.CSharp.Tests/`
  - `tests/Deedle.PerfTests/`

### Build without R

If R is not installed, run the `AllNoR` target. This omits `Deedle.RProvider.Plugin` from the build and uses the `AssemblyInfoNoR` / `BuildNoR` / `RunTestsNoR` chain (the test suite itself is the same):

```bash
dotnet fake build --target AllNoR
```

### Running a single target

```bash
dotnet fake build --target <TargetName>
```

Useful individual targets: `Clean`, `Build`, `BuildNoR`, `RunTests`, `RunTestsNoR`.

## CI Workflow

CI is defined in `.github/workflows/pull-requests.yml` (PRs) and `.github/workflows/push-master.yml` (pushes to `master`). Both run identical jobs on **`ubuntu-latest`** and **`windows-latest`**.

The exact CI steps are:

```yaml
- dotnet tool restore
- dotnet paket restore
- dotnet fake build          # runs the full All target
```

A PR is considered passing when both the Windows and Ubuntu jobs succeed.

## Common Failure Modes

| Symptom | Likely cause |
|---|---|
| Build fails immediately with "fake not found" | `dotnet tool restore` was not run |
| Paket errors about missing packages | `dotnet paket restore` was not run |
| `Deedle.RProvider.Plugin` compile errors | R is not installed or `R_HOME` is unset |
| Test failures in `Deedle.Tests` | Logic regression in `src/Deedle/` |
| Test failures in `Deedle.Math.Tests` | Logic regression in `src/Deedle.Math/` |
| AssemblyInfo file has wrong version | `RELEASE_NOTES.md` header format is invalid |

## Code Style and Conventions

- All source is **F#** (`.fs`) except `Deedle.CSharp.Tests` which is C#.
- Target frameworks: `netstandard2.0`, `netstandard2.1`, `net461`, and `net5.0` depending on the project.
- Version is driven by `RELEASE_NOTES.md` — keep the top entry well-formed (`## x.y.z - YYYY-MM-DD`) so FAKE can parse it for `AssemblyInfo` generation.
- Do not manually edit `src/Deedle/Common/AssemblyInfo.fs` or any other `AssemblyInfo.fs`; they are auto-generated by the build.
